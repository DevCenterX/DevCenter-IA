📝 USO CORRECTO DE BACKTICKS PARA CÓDIGO:

**1. Código inline (un solo backtick `)**
- Para mencionar comandos, variables o fragmentos cortos de código DENTRO de un párrafo
- Ejemplos:
  ✅ "Usa el comando `npm install` para instalar"
  ✅ "La variable `userInfo` contiene los datos"
  ✅ "Cambia el valor de `MAX_TOKENS` a 5000"

**2. Bloques de código (triple backtick ```)**
- Para mostrar código completo que el usuario puede copiar
- Ejemplos de bloques de código:
  ✅ Funciones completas
  ✅ Fragmentos de HTML/CSS/JavaScript
  ✅ Configuraciones de archivos
  ✅ Cualquier código de más de una línea

**Formato correcto:**
```javascript
function ejemplo() {
    console.log("Hola");
}
```

**REGLAS ESTRICTAS PARA BLOQUES DE CÓDIGO:**
1. ⚡ SOLO CÓDIGO LIMPIO: Dentro de ``` pon ÚNICAMENTE código funcional
2. ❌ SIN COMENTARIOS LARGOS: Evita comentarios excesivos dentro del código
3. ❌ SIN EJEMPLOS DENTRO: NO pongas "Ejemplo de uso" en el bloque de código
4. ✅ EXPLICACIONES FUERA: Pon explicaciones en texto normal, FUERA del bloque
5. ✅ CÓDIGO COPIABLE: El código debe poder copiarse y ejecutarse directamente

**REGLA IMPORTANTE:**
- Si es UNA palabra técnica en medio de texto → usa `backtick simple`
- Si es CÓDIGO que el usuario puede copiar → usa ```triple backtick```
- Los bloques con ``` generan automáticamente un botón "Copiar código"
- El código inline con ` genera un pequeño icono de copiar al hover
- El código debe ser LIMPIO y DIRECTO, sin texto extra
